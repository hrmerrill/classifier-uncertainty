# classifier-uncertainty — agent instructions

## Development workflow

Package manager is `uv`. All commands run via `uv run <tool>`.

| Task | Command |
|---|---|
| Format + lint | `make format` |
| Type / docstring / lint checks | `make check` |
| Tests with coverage | `make test` |
| Build docs | `make docs` |
| Serve docs locally | `make docs-serve` |

**Always run `make check && make test` after any code change before reporting done.**
Run `make docs` when touching `docs/`, `hooks.py`, or plot methods — verify the build is clean.

## Check suite details

`make check` runs three tools in sequence:

1. **ruff** — formatting and linting (`line-length = 100`, rules E/F/W/I)
2. **ty** — type checking. Does not respect `# type: ignore[attr-defined]`; use `assert value is not None` to narrow `T | None` types instead.
3. **interrogate** — 100% docstring coverage, including **inner functions** and single-underscore private methods. Every `def` that interrogate counts (check with `uv run interrogate -vv src/`) needs a docstring.

`make test` requires ≥ 90% line coverage. Plot methods and branching code paths must be exercised.

## Docstrings

Use **numpy style** throughout — required for mkdocstrings to render the API correctly.

```python
def method(self, param: float) -> MetricResult:
    """One-line summary.

    Parameters
    ----------
    param : float
        Description.

    Returns
    -------
    MetricResult
        Description.

    Raises
    ------
    ValueError
        When param is out of range.
    """
```

Class-level parameters belong in the class docstring `Parameters` section (not `__init__`, which is ignored by interrogate via `ignore-init-method = true`).

## Testing conventions

- Set `matplotlib.use("Agg")` at the **top of the test file**, before any `import matplotlib.pyplot`. Placing it inside a fixture or test function is too late on some backends.
- Test every plot method (`result.plot()`, `curve.plot()`); these are real code paths and coverage will fail without them.
- Import private submodules directly when testing internal behaviour: `from classifier_uncertainty._sampler import CMSampler`.

## Documentation

Figures are generated by `hooks.py` (repo root) using the mkdocs `on_post_build` hook.

**Use `on_post_build`, never `on_pre_build`, for any hook that writes files.**
`on_pre_build` writes into `docs/`, which mkdocs watches; every build overwrites the files, the watcher detects the changes, and triggers another build — an infinite loop. `on_post_build` writes directly to `config["site_dir"]`, which is not watched.

Figures are generated by `hooks.py` via two hooks:

- **`on_pre_build`**: writes to `docs/assets/` **only if absent**. mkdocs transforms image paths (e.g. `assets/foo.png` → `../assets/foo.png` for subpages) only when the file exists in `docs/`, so this write is required for correct HTML. Skipping on subsequent runs prevents an infinite rebuild loop during `mkdocs serve`.
- **`on_post_build`**: always writes to `site/assets/` to keep the served site current.

To force regeneration after editing `hooks.py`: `rm -rf docs/assets/ && make docs`

## numpy compatibility

numpy 2.0 removed `np.trapz`. Use this pattern for cross-version compatibility:

```python
fn = getattr(np, "trapezoid", None) or getattr(np, "trapz", None)
assert fn is not None  # narrows type for ty; also a runtime guard
result = fn(y, x, axis=0)
```

## Plotting gotchas

**Threshold grid ordering.** `_threshold_grid` returns `[max_score+1, min_score, q1…q(n-3), min_score-1]` — the first entry is all-negative (x=0) and the second is all-positive (x=1). Passing this directly to `ax.plot()` draws diagonal slashes across the figure. Always sort by the x-axis before connecting points:

```python
order = np.argsort(mean_x)
ax.plot(mean_x[order], mean_y[order], ...)
```

This applies to any curve that sweeps thresholds (ROC, PR). The `auc` property already sorts per-sample via `np.argsort(..., axis=0)` — that is correct and separate.
